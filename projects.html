My Projects

Throughout my university years, I had the opportunity to participate in projects integrating multiple concepts, including networking, computer science, and electronics. I thrived in these projects, pushing my skills to their limits, and here's a summary:

Connected Greenhouse

The goal of this project was to combine and apply the various skills and knowledge acquired during the school year. We decided to create a connected greenhouse system, based on a master-slave relationship between a Raspberry Pi, acting as master, and Arduino devices connected to the Raspberry Pi using the ISM band provided by the NRF24L01 electronic module.

Each Arduino device was autonomous, capable of reaching a range of up to 1 km, making this solution suitable for large-scale implementation. Each device was equipped with a battery with a one-week autonomy, a humidity and temperature sensor, as well as a motion sensor allowing the integration of an alarm system. The devices sent various data to the Raspberry Pi every second, which then acted as an aggregator, redirecting the data flow to a remote MQTT server. On this server, a solution based on PHP and Ajax was installed to dynamically display the information acquired from the different Arduino devices on a Chart.JS graph.

This project was a great personal and professional success, and a lot of knowledge was acquired, particularly regarding radio wave data transmission and MQTT server implementation, as well as all the potential offered by this solution.

RC Car Project

This project, like the previous one, aimed to consolidate and apply as much knowledge and skills acquired in our courses as possible. However, this time, we wanted to push our limits even further. The previous project had been interrupted and had not progressed beyond the prototype stage on a breadboard due to hardware and circumstantial constraints. This time, we decided to push the project as far as possible and create the most professional solution possible, this time on a printed circuit board (PCB).

We chose to build a remote-controlled car, but not just any remote-controlled car; we wanted to get as close as possible to a Formula 1 car model. That's why the car is equipped with various sensors to analyze its behavior. The car chassis is a combination of salvaged and improved components, capable of reaching a maximum speed of 100 km/h. The remote control is a Chinese product bought online, of which we only used the shell, and we created a custom printed circuit board (PCB) with the same dimensions to fit inside the remote control. This allowed us to use the remote control for our project with our own PCB. Communication is based on the NRF24L01 module, allowing a communication distance of more than 1 km in optimal conditions.

To deepen our knowledge, we based the project on an STM32 F103, one of the best-documented and most easily accessible STM microcontrollers online. To analyze the various data, we set up a web server allowing real-time display and historical storage of collected statistics. This information was collected and received by a Java socket server, which served as an aggregator for the different devices. Each device, i.e., each remote control, sent the statistics received from the car to the server, either through a 2G module with a SIM card provided by the user, or via an Android application that served as a bridge between the remote control and the socket server. The application connected to the remote control via Bluetooth.

This project is a source of pride for us, as the car still exists and is displayed as a trophy. A lot of new knowledge was acquired, and a lot of passion was invested in this project, which took six months of work. We are proud to have completed this project successfully.

SAE 501: Design, Implement and Present a Technical Solution

This project aims to deepen the understanding and practical application of mobile communication technologies, specifically focusing on setting up an LTE network using Open5GS and srsRAN tools. The goal is to build a functional LTE network, allowing exploration of the different components and configurations necessary for effective mobile communication. This project allowed me to discover new hardware and work processes, such as the USRP and the programmable SIM card reader. It also allowed me to discover in more detail how LTE works and identification in a 4G network via KI and OPC keys.

SAE 502: Project Management

This project aims to provide a practical case of deployment by an IT expert of a dockerized solution containing three containers:

  • Apache: Web server with PHP 8.
  • MySQL: Database.
  • PhpMyAdmin: Database management tool.
For this, I used a single docker-compose file format to facilitate deployment.

The goals I set for myself on this project are:

  • Uniformity → Each phase of the project benefits from a consistent environment, eliminating compatibility issues between computers.
  • Ease of deployment → With Docker, setting up the development environment is quick and simplified.
  • Isolation → Docker containers operate independently, reducing conflicts between services.
  • Portability → Docker ensures that the application works the same way on all systems where it is deployed.
This consistency throughout the project helped maintain the stability and reliability of the application throughout the development process.
